# 成员 B 最终验收 Todo List

目标：在中期已经完成时间轴和参战方共现网络的基础上，进一步提升最终系统的分析能力、交互完整性和可视化表达质量，让时间轴、网络图、地图、统计面板之间形成更清晰、可解释、可演示的分析链路。

## 一、需要新增或补强的功能

- [ ] 增加网络图的选中 participant 聚焦模式。
  - 当前选中的 participant 如果不在 Top 20 节点中，只显示提示；最终版建议让选中节点强制进入网络，或切换为以该 participant 为中心的一阶邻居 ego network。
- [ ] 增加网络边的详情信息。
  - 鼠标悬停或点击边时，显示两个 participant 的共现次数、共现年份范围和若干代表性 conflict events。
- [ ] 增加网络节点的详情信息。
  - 鼠标悬停或点击节点时，显示该 participant 的事件数量、涉及 conflict groups、活跃年份范围和代表事件。
- [ ] 增加时间轴年度详情摘要。
  - 点击或悬停某一年时，除切换地图年份外，展示该年事件总数、当前筛选命中数、主要 conflict groups 和 Top participants。
- [ ] 增加时间轴区间对比能力。
  - 支持比较两个年份或两个阶段的事件数量变化，例如一战前后、二战前后、冷战早期与后期。
- [ ] 增加 case study 内部的时间轴和网络解释。
  - 对 World War I、World War II 等典型案例补充可解释的图上模式，例如事件高峰年份、核心 participant、强共现关系。
- [ ] 增加空结果时的推荐操作。
  - 当筛选后时间轴、网络或地图为空时，提示用户放宽年份范围、清除 participant 或切换 conflict group。
- [ ] 增加交互状态的可复位能力。
  - 网络节点选中、时间轴年份选中、地图事件选中应能通过清除按钮或再次点击恢复到默认状态。

## 二、需要完善的交互方式

- [ ] 优化时间轴与地图的联动反馈。
  - 点击时间轴年份后，地图事件点和 CShapes 边界快照已经会变化；最终版需要让变化更明显，例如短暂高亮当前年份、显示快照年份说明。
- [ ] 优化时间轴事件列表。
  - 当前最多显示 24 条事件；建议增加“查看更多”或分页，避免高事件年份的信息被截断。
- [ ] 优化网络节点点击后的联动逻辑。
  - 点击节点后不仅更新全局 participant 筛选，还应在网络图内保留上下文，突出相关边和相邻节点，弱化无关部分。
- [ ] 优化选中 participant 不在网络可见范围时的体验。
  - 当前只提示 outside Top 20；建议直接显示该 participant 的局部网络，或提供“show selected participant”按钮。
- [ ] 优化筛选器变更后的状态清理。
  - 当 conflict group、年份范围或 participant 改变后，检查当前 selected event 是否仍然有效；无效时应清空详情面板并给出明确状态。
- [ ] 优化键盘与可访问性交互。
  - 网络节点目前支持 Enter/Space；还可以补充焦点样式、aria 描述和时间轴柱子的键盘选择。
- [ ] 增加 tooltip 或图例解释。
  - 时间轴的双层柱、网络节点大小、边宽、选中颜色应在图例中保持清楚一致，减少答辩时口头解释成本。
- [ ] 增加手动验证路径。
  - 固定 2-3 条最终演示流程，例如“选择 World War II -> 点击 1944 -> 点击 United Kingdom -> 查看网络和地图变化”。

## 三、需要优化的可视化问题

- [ ] 优化网络图布局。
  - 当前圆形布局稳定但容易产生长边交叉；最终版可尝试分层圆形布局、按社群分组布局或轻量 force layout。
- [ ] 优化网络标签密度。
  - Top 20 节点全部显示标签时容易拥挤；可以只显示高权重节点标签，悬停显示完整名称，或避让重叠标签。
- [ ] 优化网络边的视觉层级。
  - 强关系边应更突出，弱关系边应更淡；选中节点相关边应高亮，无关边应降低透明度。
- [ ] 优化网络节点颜色编码。
  - 可以用颜色区分国家、帝国、联盟或其他 actor 类型；若数据质量不足，则至少保持选中态、普通态、弱化态清晰。
- [ ] 优化时间轴极值年份显示。
  - 当某些年份事件数很高时，其他年份柱子会显得过低；可考虑对高度使用平方根缩放或提供局部缩放。
- [ ] 优化时间轴横向阅读。
  - 年份范围长时，标签间隔、滚动位置和当前年份定位需要稳定，避免用户失去当前所在年份。
- [ ] 优化颜色一致性。
  - 时间轴、网络图和地图中“选中”“筛选命中”“背景上下文”的颜色语义应尽量统一。
- [ ] 优化移动端或窄屏表现。
  - 检查时间轴、网络图、图例和说明文本在窄屏下是否重叠、溢出或难以点击。
